# 三、接口清单建议

## 1. admission 接口
### 家属端
- `GET /admission/family-visit-slots`：查询可预约时段
- `POST /admission/family-visit-reservations`：提交预约申请
- `GET /admission/family-visit-reservations/my`：查看我的预约记录

### 医护端
- `GET /admission/family-visit-reservations`：预约审核列表
- `POST /admission/family-visit-reservations/{id}/approve`：审核通过
- `POST /admission/family-visit-reservations/{id}/reject`：审核拒绝
- `GET /admission/assessment-records`：评估记录列表
- `POST /admission/assessment-records`：创建入院评估
- `POST /admission/assessment-records/{id}/complete`：完成评估

---

## 2. contract 接口
### 医护端
- `POST /contract/online-contracts`：上传合同并发起签字
- `GET /contract/online-contracts`：合同列表
- `GET /contract/online-contracts/{id}`：合同详情

### 家属端
- `GET /contract/online-contracts/my`：我的待签/已签合同
- `POST /contract/online-contracts/{id}/sign`：在线签字

---

## 3. health 接口
### 医护端
- `GET /health/assessments`：健康评估列表
- `POST /health/nurse-assessments`：提交护士评估
- `POST /health/doctor-assessments`：提交医生评估
- `GET /health/doctor-round-records`：查房记录列表
- `POST /health/doctor-round-records`：新增查房记录

### 家属端
- `GET /health/profile/{elderId}`：查看老人健康档案
- `GET /health/nurse-assessments/{elderId}`：查看健康评估记录

## 4. caredelivery 接口

- `POST /care-delivery/service-templates`：新增服务模板
- `GET /care-delivery/service-plans`：老人服务清单列表
- `POST /care-delivery/service-plans`：创建老人服务清单
- `GET /care-delivery/nurse-care-records`：护理记录列表
- `POST /care-delivery/nurse-care-records`：新增护理记录

### 护理员端
- `GET /care-delivery/my-tasks`：我的服务任务
- `POST /care-delivery/my-tasks/{id}/check-in`：服务打卡
- `GET /care-delivery/my-check-ins`：我的打卡记录

### 家属端
- `GET /care-delivery/family/service-plans/{elderId}`：查看服务清单
- `GET /care-delivery/family/check-ins/{elderId}`：查看护理打卡记录

# 五、当前最需要你继续确认的点

1. 家属端、护理员端是否真的共用一个前端应用，只通过角色区分菜单与页面
2. 预约参观是否需要取消 / 改期能力
3. 云签字是否只做“图片签字”，还是需要电子签服务对接
4. 健康评估、护理记录、查房记录的表单模板是否已有现成版本
5. 护理员季度考核是否由系统发题，还是仅记录外部考核结果
6. 风险预警规则是平台内判断，还是完全由外部手环系统推送
