# 医养结合智慧养护平台 - 阶段四表结构草案

> 范围：培训视频、培训记录、季度考核、满意度测评、综合考评。

---

# 一、建表原则

- 阶段四主要落在 `quality` 域
- 问卷和题目结构阶段四先允许使用 JSON 保存
- 统计结果单独落表，避免每次动态聚合计算

---

# 二、quality 域表结构

## 1. training_video
用途：培训视频主表

### 建议字段
- `id` bigint PK
- `title` varchar(128) NOT NULL
- `video_url` varchar(255) NOT NULL
- `duration_seconds` int NOT NULL
- `enabled` boolean NOT NULL
- `created_at` datetime NOT NULL
- `updated_at` datetime NOT NULL

### 建议索引
- `(enabled)`
- `(title)`

---

## 2. caregiver_training_record
用途：护理员培训观看记录

### 建议字段
- `id` bigint PK
- `caregiver_id` bigint NOT NULL
- `training_video_id` bigint NOT NULL
- `progress_percent` int NOT NULL
- `completed` boolean NOT NULL
- `completed_at` datetime NULL
- `created_at` datetime NOT NULL
- `updated_at` datetime NOT NULL

### 建议索引
- `(caregiver_id, completed)`
- `(training_video_id)`
- `(caregiver_id, training_video_id)` unique

---

## 3. caregiver_quarterly_exam
用途：季度考核主表

### 建议字段
- `id` bigint PK
- `caregiver_id` bigint NOT NULL
- `quarter_label` varchar(32) NOT NULL
- `questionnaire_data` text NULL
- `result_status` varchar(32) NOT NULL
- `score` decimal(5,2) NULL
- `review_comment` varchar(255) NULL
- `reviewed_by` varchar(64) NULL
- `reviewed_at` datetime NULL
- `created_at` datetime NOT NULL
- `updated_at` datetime NOT NULL

### 建议索引
- `(caregiver_id, quarter_label)`
- `(quarter_label, result_status)`

### 建议约束
- 同一护理员同一季度建议只保留一条主考核记录

---

## 4. family_satisfaction_survey
用途：家属满意度测评

### 建议字段
- `id` bigint PK
- `elder_id` bigint NOT NULL
- `family_user_id` bigint NOT NULL
- `caregiver_id` bigint NULL
- `service_period_start` date NOT NULL
- `service_period_end` date NOT NULL
- `survey_data` text NOT NULL
- `score` decimal(5,2) NOT NULL
- `submitted_at` datetime NOT NULL
- `created_at` datetime NOT NULL
- `updated_at` datetime NOT NULL

### 建议索引
- `(elder_id, submitted_at desc)`
- `(caregiver_id, submitted_at desc)`
- `(family_user_id, submitted_at desc)`

### 建议约束
- 同一家属 + 同一老人 + 同一服务周期，阶段四建议只允许一条有效测评

---

## 5. caregiver_performance_summary
用途：护理员综合考评结果

### 建议字段
- `id` bigint PK
- `caregiver_id` bigint NOT NULL
- `period_label` varchar(32) NOT NULL
- `exam_score` decimal(5,2) NULL
- `family_satisfaction_score` decimal(5,2) NULL
- `overall_score` decimal(5,2) NOT NULL
- `result_level` varchar(32) NOT NULL
- `generated_at` datetime NOT NULL
- `created_at` datetime NOT NULL
- `updated_at` datetime NOT NULL

### 建议索引
- `(caregiver_id, period_label)`
- `(period_label, result_level)`

### 建议约束
- 同一护理员同一周期建议只保留一条综合考评结果

---

### 新增 taskType

### 新增日志 event

- `QUARTERLY_EXAM_CREATED`
- `QUARTERLY_EXAM_REVIEWED`

# 四、业务关系说明

## 1. 培训视频
- `training_video` 定义培训内容
- `caregiver_training_record` 记录每位护理员观看状态

## 2. 季度考核
- `caregiver_quarterly_exam` 记录某个护理员某个季度的考核结果

## 3. 家属满意度
- `family_satisfaction_survey` 记录家属对服务周期的评价
- 可选与护理员绑定，也可只与老人绑定（待业务最终确认）

## 4. 综合考评
- `caregiver_performance_summary` 保存计算后的结果
- 数据来源于考核成绩 + 满意度评分

---

# 五、阶段四建表优先级

## P0
1. `training_video`
2. `caregiver_training_record`
3. `caregiver_quarterly_exam`
4. `family_satisfaction_survey`
5. `caregiver_performance_summary`

## P1
1. 问卷模板表
2. 问卷题目子表
3. 培训分类表
4. 考评权重配置表

# 六、后续如需增强的表结构方向

## 1. 问卷模板化
若后续满意度和季度考核都要求灵活配置，建议补：
- `questionnaire_template`
- `questionnaire_template_item`

## 2. 培训分类
若培训内容增多，建议补：
- `training_category`

## 3. 评分权重配置
若综合考评需要后台可调权重，建议补：
- `performance_weight_config`

---

# 七、阶段四仍待确认的表结构问题

1. 一个护理员一个季度是否允许多次考核版本
2. 满意度是否必须绑定护理员
3. 综合考评周期按季度、月度还是自定义周期
4. 培训是否需要记录观看开始时间、最后播放位置
5. 是否需要保留问卷原始明细用于审计
