# 医养结合智慧养护平台 - 需求整理文档

> 目标：把当前口述需求整理成可 review 的产品需求初稿，便于后续拆分页面、流程和接口。

---

## 一、产品定位

面向医院 / 医养机构的智慧养护管理平台，支持三端联动：

- **医护端（Web 管理端）**：医生、护士、超级管理员使用
- **护理员端（APP）**：持证护理人员使用
- **家属端（APP）**：患者 / 老人家属使用

目标是把入院评估、日常养护、智能监测、远程探视、线上管理、考核评价等流程数字化。

---

## 二、核心角色

### 1. 医护端角色
- 超级管理员：机构账户管理、人员管理
- 护士：入院评估、每日护理记录、审核相关工作
- 医生：查房记录、健康评估、风险判断

### 2. 护理员端角色
- 持证上岗护理人员
- 需先完成资质审核，再登录使用

### 3. 家属端角色
- 老人 / 患者家属
- 关注入院流程、服务情况、探视、评价与健康信息

---

## 三、三端功能需求

## （一）家属端 APP

### 1. 机构信息展示
- 展示医养宣传视频
- 展示服务清单
- 展示医院 / 机构资质介绍

### 2. 实时知情与远程服务
- 查看院内情况
- 减少线下探视依赖
- 云探视：支持与护理员视频通话（待定）
- 查看每日服务清单
- 查看护理打卡记录

### 3. 评价与互动
- 月度满意度测评自动提示（仅服务周期内）
- 问卷表格支持自主上传

### 4. 预约与签约
- 预约参观：
  - 可选时段为 8:00 - 17:00
  - 每半小时 1 人
  - 提交后需医护端审核
- 云签字：
  - 医护端上传合同
  - 家属端 APP 在线签字确认

### 5. 健康信息查看
- 查看入院健康评估
- 查看护理记录
- 查看查房记录等相关信息

---

## （二）护理员端 APP

### 1. 注册与资质审核
- 注册时上传：
  - 医疗护理员证（图片）
  - 身份证
- 资质提交后由医护端审核
- 审核通过后方可注册使用

### 2. 日常工作
- 每日打卡
- 勾选完成当日服务清单

### 3. 培训与考核
- 线上培训：观看培训视频
- 查看个人考核结果
- 展示考核通过状态

### 4. 云探视
- 响应家属端视频通话请求（待定）

---

## （三）医护端 Web / 电脑端

### 1. 人员管理
- 超级管理员账户管理页面
- 手动录入 / 管理护士、医生账号
- 护理员资质审核：审核护理员上传的证件资料

### 2. 培训与考核管理
- 查看护理员培训视频播放完成记录
- 季度考核审核：
  - 问卷形式
  - 记录操作人
  - 判定是否完成
- 护理员满意度测评结果手动录入
- 个人考评依据：
  - 考核成绩
  - 家属满意度（待完善）

### 3. 入院与健康管理
- 入院健康评估
- 对接医院系统获取既往病历（待定）
- 替代原健康体检表
- 评估流程：
  - 家属签合同后启动
  - 护士每周 1 次评估
  - 医生评估
  - 生成评估总结
  - 判定老人类型（全失能 / 半失能）

### 4. 日常养护记录
- 病床信息管理：表格形式录入 / 管理
- 护士填写每日护理记录（表格）
- 护士支持自定义服务清单（模板 + 个性化添加）
- 医生每日填写查房记录（文字形式）
- 待办提醒：
  - 查房 / 记录不达标进入待处理清单
  - 显示危险符号预警

### 5. 家属端业务审核
- 审核家属端“预约参观”申请
- 上传合同至家属端
- 发起云签字流程

### 6. 预警与安全
- 风险预警机制（待定）
- 对接手环预警系统，接收健康 / 安全预警

- 调取既往病历（待定）

---

## 五、核心流程

### 1. 入院流程
1. 家属预约参观
2. 医护端审核通过
3. 签约（云签字）
4. 入院健康评估
5. 判定失能等级
6. 制定服务清单

### 2. 日常养护流程
1. 护士 / 医生每日记录
2. 护理员打卡执行
3. 家属实时查看
4. 月度满意度评价

### 3. 护理员管理流程
1. 护理员注册并上传资质
2. 医护端审核通过
3. 线上培训
4. 季度考核
5. 日常考评

### 4. 考核机制流程
1. 季度问卷考核
2. 家属满意度统计
3. 护士录入结果
4. 形成个人综合考评
