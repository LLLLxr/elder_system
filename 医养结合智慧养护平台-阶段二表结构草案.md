# 医养结合智慧养护平台 - 阶段二表结构草案

> 范围：支撑云签字、入院健康评估、服务模板与老人服务清单配置。

---

# 一、建表原则

- 继续沿用状态字符串 + 审计时间字段模式
- 跨域流程不在主表硬编码过多流程信息，统一通过 orchestration 关联
- 阶段二先允许部分结构使用 JSON，后续稳定后再拆子表

---

# 二、contract 域表结构

## 1. online_contract_record
用途：云签字合同主表

### 建议字段
- `id` bigint PK
- `elder_id` bigint NOT NULL
- `family_user_id` bigint NOT NULL
- `contract_name` varchar(128) NOT NULL
- `contract_file_url` varchar(255) NOT NULL
- `remark` varchar(255) NULL
- `sign_status` varchar(32) NOT NULL
- `initiated_by` varchar(64) NOT NULL
- `initiated_at` datetime NOT NULL
- `signed_at` datetime NULL
- `cancelled_at` datetime NULL
- `created_at` datetime NOT NULL
- `updated_at` datetime NOT NULL

### 建议索引
- `(family_user_id, sign_status)`
- `(elder_id, sign_status)`
- `(sign_status, initiated_at desc)`

---

## 2. contract_sign_log
用途：合同签署日志

### 建议字段
- `id` bigint PK
- `contract_id` bigint NOT NULL
- `operator_user_id` bigint NULL
- `event` varchar(64) NOT NULL
- `signature_image_url` varchar(255) NULL
- `remark` varchar(255) NULL
- `created_at` datetime NOT NULL

### 建议 event
- `INITIATED`
- `SIGNED`
- `CANCELLED`

---

# 六、业务关系说明

## 1. 合同签字
- `online_contract_record` 为主表
- 签字过程写入 `contract_sign_log`
- 同时在 orchestration 中维护待办与日志

## 2. 入院健康评估
- `admission_health_assessment_record` 为评估主表
- 护士、医生评估分别写入子记录表
- 评估完成后更新 `elder_health_profile`

## 3. 服务清单
- `care_service_template` 提供标准模板
- `elder_service_plan` 保存某位老人的当前生效清单

# 八、阶段二仍待确认的表结构问题

1. 合同是否需要版本管理
2. 健康评估主表是否需要记录护士与医生具体负责人
3. 失能等级枚举是否会继续扩展
4. 服务清单项是否要做标准编码字典
5. `elder_health_profile` 是否需要同步保存医院侧病历摘要
