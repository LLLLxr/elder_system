# 医养结合智慧养护平台 - 阶段一接口字段明细

> 范围：阶段一只覆盖“家属预约参观”和“护理员资质申请/审核”两个最小闭环，以及它们需要的基础查询接口。

---

# 一、接口设计约定

- 鉴权：沿用现有 JWT + Gateway 头透传机制
- 分页：沿用现有分页响应结构
- 时间字段统一使用 ISO 时间字符串
- 状态字段统一使用字符串枚举
- 列表查询接口优先支持分页与基础筛选

---

# 二、admission 域接口

## 1. 查询可预约时段
### `GET /admission/family-visit-slots`

### Query 参数
- `visitDate`：string，必填，格式 `YYYY-MM-DD`

### Response
```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "timeSlot": "08:00-08:30",
      "available": true,
      "reservedCount": 0,
      "capacity": 1
    }
  ]
}
```

### 字段说明
- `timeSlot`：时间段
- `available`：是否可预约
- `reservedCount`：已预约人数
- `capacity`：时段容量，当前固定 1

---

## 2. 家属提交预约参观申请
### `POST /admission/family-visit-reservations`

### Request Body
```json
{
  "elderId": 10001,
  "contactName": "张三",
  "contactPhone": "13800000000",
  "visitDate": "2026-05-10",
  "visitTimeSlot": "08:00-08:30",
  "remark": "第一次参观，想了解护理环境"
}
```

### 字段说明
- `elderId`：老人ID，可选；如果还未绑定老人，可后续改为空或支持预登记
- `contactName`：联系人姓名
- `contactPhone`：联系电话
- `visitDate`：预约日期
- `visitTimeSlot`：预约时段
- `remark`：备注，可选

### Response
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "reservationId": 9001,
    "status": "PENDING",
    "createdAt": "2026-05-03T10:00:00"
  }
}
```

---

## 3. 家属查看我的预约记录
### `GET /admission/family-visit-reservations/my`

### Query 参数
- `page`：int，默认 0
- `size`：int，默认 10
- `status`：string，可选

### Response
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "content": [
      {
        "reservationId": 9001,
        "elderId": 10001,
        "contactName": "张三",
        "contactPhone": "13800000000",
        "visitDate": "2026-05-10",
        "visitTimeSlot": "08:00-08:30",
        "status": "PENDING",
        "reviewComment": null,
        "createdAt": "2026-05-03T10:00:00"
      }
    ],
    "page": 0,
    "size": 10,
    "totalElements": 1,
    "totalPages": 1
  }
}
```

---

## 4. 医护端预约审核列表
### `GET /admission/family-visit-reservations`

### Query 参数
- `page`：int，默认 0
- `size`：int，默认 20
- `status`：string，可选
- `visitDate`：string，可选
- `contactName`：string，可选
- `contactPhone`：string，可选

### Response 字段
列表项建议包含：
- `reservationId`
- `elderId`
- `familyUserId`
- `contactName`
- `contactPhone`
- `visitDate`
- `visitTimeSlot`
- `status`
- `remark`
- `reviewBy`
- `reviewTime`
- `reviewComment`
- `createdAt`

---

## 5. 医护端预约详情
### `GET /admission/family-visit-reservations/{id}`

### Response 字段
- 基本信息
- 审核信息
- 流程任务信息（可选）

---

## 6. 医护端审核通过预约
### `POST /admission/family-visit-reservations/{id}/approve`

### Request Body
```json
{
  "reviewComment": "预约通过，请按时到院"
}
```

### Response
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "reservationId": 9001,
    "status": "APPROVED",
    "reviewTime": "2026-05-03T12:00:00"
  }
}
```

---

## 7. 医护端审核拒绝预约
### `POST /admission/family-visit-reservations/{id}/reject`

### Request Body
```json
{
  "reviewComment": "当前时段已满，请重新预约"
}
```

### Response
- 与 approve 类似，只是 `status=REJECTED`

# 四、orchestration 域接口（阶段一最小集）

## 1. 查询待办任务列表
### `GET /care-orchestration/workflow-tasks`

### Query 参数
- `taskType`：string，可选
- `status`：string，可选
- `assigneeRole`：string，可选
- `page`：int
- `size`：int

### 阶段一 taskType 建议值
- `FAMILY_VISIT_RESERVATION_REVIEW`
- `CAREGIVER_QUALIFICATION_REVIEW`

### Response 列表字段
- `taskId`
- `taskType`
- `businessId`
- `businessType`
- `status`
- `assigneeRole`
- `title`
- `summary`
- `dueAt`
- `createdAt`

---

## 2. 查询单个业务的流程日志
### `GET /care-orchestration/workflow-transition-logs`

### Query 参数
- `businessType`
- `businessId`

### Response 字段
- `logId`
- `businessType`
- `businessId`
- `fromState`
- `toState`
- `event`
- `reason`
- `createdBy`
- `createdAt`
