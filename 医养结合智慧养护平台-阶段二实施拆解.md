# 医养结合智慧养护平台 - 阶段二实施拆解

> 阶段二目标：从“准入流程”进入“正式在服前准备流程”，把云签字、入院健康评估、服务清单配置三块打通。

---

# 一、阶段二目标

在阶段一已经完成“预约参观”和“护理员资质审核”闭环的基础上，阶段二重点完成：

1. 云签字闭环
2. 入院健康评估闭环
3. 老人服务清单配置闭环
4. 从签约到在服准备的流程串联

阶段二做完后，应形成以下业务路径：

- 家属预约参观 → 审核通过
- 医护端上传合同并发起云签字
- 家属完成在线签字
- 医护端发起入院健康评估
- 护士 / 医生完成评估
- 生成失能等级/评估结论
- 医护端为老人配置服务清单

---

# 二、阶段二功能范围

## 1. contract 域
负责：
- 合同上传
- 发起云签字
- 家属查看待签合同
- 家属在线签字
- 查看签署状态

## 2. health 域
负责：
- 入院健康评估主记录
- 护士评估
- 医生评估
- 评估总结
- 失能等级判定
- 健康档案更新

## 3. caredelivery 域
负责：
- 服务模板
- 老人个性化服务清单
- 服务清单启用状态

## 4. careorchestration 域
负责：
- 云签字待办
- 健康评估待办
- 服务清单配置待办
- 签约 → 评估 → 服务准备的流程日志

# 五、阶段二开发任务清单

## A. 云签字闭环
### A1. 医护端
- 上传合同并保存合同记录
- 发起签字任务
- 查询合同状态

### A2. 家属端
- 查看我的待签合同
- 查看合同详情
- 完成在线签字

### A3. orchestration
- 发起签字时生成待办或流程节点
- 家属签字后更新流程状态

验收标准：
- 医护端发起签字后，家属端可见待签合同
- 家属签字后，合同状态变更为已签

---

## B. 健康评估闭环
### B1. 医护端评估主记录
- 创建入院评估主记录
- 绑定老人、合同、家属信息

### B2. 护士评估
- 护士填写表单
- 保存评估结果

### B3. 医生评估
- 医生填写评估表单
- 保存结果

### B4. 评估总结
- 生成评估总结
- 给出失能等级（全失能 / 半失能，后续可扩）
- 更新老人健康档案

### B5. orchestration
- 创建健康评估待办
- 护士完成后进入医生待办
- 医生完成后进入服务清单配置待办

验收标准：
- 护士/医生评估可以分步完成
- 最终能得到评估结果和失能等级

---

## C. 服务清单配置闭环
### C1. 服务模板
- 新增服务模板
- 查看模板列表
- 启停模板

### C2. 老人服务清单
- 为单个老人配置服务清单
- 支持“模板选择 + 个性化调整”
- 保存为老人当前生效服务清单

### C3. orchestration
- 健康评估完成后生成服务清单配置待办
- 配置完成后流程进入“在服准备完成”或下一状态

验收标准：
- 每个老人能有一份当前有效服务清单
- 清单来源可追溯到模板或个性化配置

---

# 六、阶段二推荐落地顺序

1. contract 表与接口
2. 家属端签字页面
3. 医护端合同发起页
4. health 评估主记录 + 护士评估
5. 医生评估 + 评估总结
6. caredelivery 服务模板
7. 老人服务清单配置
8. orchestration 流程串联

---

# 七、阶段二完成标准

满足以下条件即可视为阶段二完成：

1. 医护端可发起合同签字
2. 家属端可完成在线签字
3. 医护端可创建并完成入院健康评估
4. 系统能生成失能等级与评估总结
5. 医护端可为老人配置服务清单
6. 签约 → 评估 → 服务清单流程可串通

---

# 八、阶段二仍待确认项

1. 在线签字是画板签字、上传图片，还是第三方电子签
2. 健康评估表单字段是否已有现成模板
3. 失能等级是否只有“全失能 / 半失能”，还是需要更多等级
4. 服务清单模板是否按老人类型区分
5. 健康评估完成后是否自动进入在服状态，还是服务清单配置后才进入
