# 医养结合智慧养护平台 - 阶段一表结构草案

> 目标：先支撑“预约参观”和“护理员资质审核”两个闭环，以及最小工作流待办。

---

# 一、建表原则

- 延续现有项目风格，状态字段使用字符串枚举
- 审核类数据保留 `review_by / review_time / review_comment`
- 所有主业务表保留 `created_at / updated_at`
- 工作流待办与业务表解耦，通过 `business_type + business_id` 关联

---

# 二、表结构草案

## 1. family_visit_reservation
用途：家属预约参观主表

### 建议字段
- `id` bigint PK
- `elder_id` bigint NULL
- `family_user_id` bigint NOT NULL
- `contact_name` varchar(64) NOT NULL
- `contact_phone` varchar(32) NOT NULL
- `visit_date` date NOT NULL
- `visit_time_slot` varchar(32) NOT NULL
- `remark` varchar(255) NULL
- `status` varchar(32) NOT NULL
- `review_by` varchar(64) NULL
- `review_time` datetime NULL
- `review_comment` varchar(255) NULL
- `created_at` datetime NOT NULL
- `updated_at` datetime NOT NULL

### 建议索引
- `(family_user_id, created_at desc)`
- `(status, visit_date)`
- `(visit_date, visit_time_slot)`

### 建议约束
- 同一日期 + 同一时段的预约数量不能超过容量上限（阶段一可先在 service 层校验）

---

## 2. caregiver_qualification_application
用途：护理员资质申请表

### 建议字段
- `id` bigint PK
- `user_id` bigint NOT NULL
- `caregiver_name` varchar(64) NOT NULL
- `id_card_no` varchar(32) NOT NULL
- `id_card_image_url` varchar(255) NOT NULL
- `certificate_image_url` varchar(255) NOT NULL
- `status` varchar(32) NOT NULL
- `review_by` varchar(64) NULL
- `review_time` datetime NULL
- `review_comment` varchar(255) NULL
- `created_at` datetime NOT NULL
- `updated_at` datetime NOT NULL

### 建议索引
- `(user_id, created_at desc)`
- `(status, created_at desc)`
- `(caregiver_name)`

### 建议约束
- 阶段一建议允许同一用户多次提交，但前端默认只展示最近一条
- 后端可增加“存在 PENDING 申请时不允许再次提交”的业务限制

---

## 3. workflow_task
用途：跨域待办任务表（如已有同类表，建议扩展而不是重复建）

### 建议字段
- `id` bigint PK
- `task_type` varchar(64) NOT NULL
- `business_type` varchar(64) NOT NULL
- `business_id` bigint NOT NULL
- `title` varchar(128) NOT NULL
- `summary` varchar(255) NULL
- `status` varchar(32) NOT NULL
- `assignee_role` varchar(64) NOT NULL
- `assignee_user_id` bigint NULL
- `due_at` datetime NULL
- `created_at` datetime NOT NULL
- `updated_at` datetime NOT NULL
- `completed_at` datetime NULL

### 建议索引
- `(task_type, status)`
- `(assignee_role, status)`
- `(business_type, business_id)`

### 阶段一 task_type 建议值
- `FAMILY_VISIT_RESERVATION_REVIEW`
- `CAREGIVER_QUALIFICATION_REVIEW`

---

## 4. workflow_transition_log
用途：流程状态流转日志（如已有同类表，建议复用/扩展）

### 建议字段
- `id` bigint PK
- `business_type` varchar(64) NOT NULL
- `business_id` bigint NOT NULL
- `from_state` varchar(64) NULL
- `to_state` varchar(64) NOT NULL
- `event` varchar(64) NOT NULL
- `reason` varchar(255) NULL
- `created_by` varchar(64) NULL
- `created_at` datetime NOT NULL

### 建议索引
- `(business_type, business_id, created_at desc)`

---

# 四、业务关系说明

## 1. 预约参观
- `family_visit_reservation` 是主业务表
- 提交预约后，在 `workflow_task` 中创建一条 `FAMILY_VISIT_RESERVATION_REVIEW`
- 审核通过/拒绝后：
  - 更新 `family_visit_reservation.status`
  - 更新 `workflow_task.status`
  - 写入 `workflow_transition_log`

## 2. 护理员资质申请
- `caregiver_qualification_application` 是主业务表
- 提交资质后，在 `workflow_task` 中创建一条 `CAREGIVER_QUALIFICATION_REVIEW`
- 审核通过/拒绝后：
  - 更新 `caregiver_qualification_application.status`
  - 更新 `workflow_task.status`
  - 写入 `workflow_transition_log`
