# 医养结合智慧养护平台 - 阶段二接口字段明细

> 范围：云签字、入院健康评估、服务清单配置。

---

# 一、contract 域接口

## 1. 医护端发起云签字
### `POST /contract/online-contracts`

### Request Body
```json
{
  "elderId": 10001,
  "familyUserId": 20001,
  "contractName": "医养服务协议",
  "contractFileUrl": "https://file.example.com/contract.pdf",
  "remark": "请尽快完成签字"
}
```

### Response
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "contractId": 5001,
    "signStatus": "PENDING_SIGN",
    "initiatedAt": "2026-05-03T14:00:00"
  }
}
```

---

## 2. 医护端查询合同列表
### `GET /contract/online-contracts`

### Query 参数
- `page`
- `size`
- `signStatus`
- `elderId`
- `familyUserId`

### 列表字段
- `contractId`
- `elderId`
- `familyUserId`
- `contractName`
- `signStatus`
- `initiatedBy`
- `initiatedAt`
- `signedAt`

---

## 3. 家属端查询我的待签/已签合同
### `GET /contract/online-contracts/my`

### Query 参数
- `page`
- `size`
- `signStatus`

### 列表字段
- `contractId`
- `elderId`
- `contractName`
- `contractFileUrl`
- `signStatus`
- `initiatedAt`
- `signedAt`

---

## 4. 家属端查看合同详情
### `GET /contract/online-contracts/{id}`

### Response 字段
- `contractId`
- `elderId`
- `familyUserId`
- `contractName`
- `contractFileUrl`
- `signStatus`
- `remark`
- `initiatedBy`
- `initiatedAt`
- `signedAt`

---

## 5. 家属端完成在线签字
### `POST /contract/online-contracts/{id}/sign`

### Request Body
```json
{
  "signatureImageUrl": "https://file.example.com/signature.png"
}
```

### Response
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "contractId": 5001,
    "signStatus": "SIGNED",
    "signedAt": "2026-05-03T15:00:00"
  }
}
```

# 二、health 域接口

## 1. 医护端创建入院健康评估主记录
### `POST /health/admission-assessments`

### Request Body
```json
{
  "elderId": 10001,
  "agreementId": 5001,
  "familyUserId": 20001
}
```

### Response
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "assessmentId": 6001,
    "assessmentStatus": "PENDING_NURSE_ASSESSMENT",
    "createdAt": "2026-05-03T16:00:00"
  }
}
```

---

## 2. 医护端查询评估列表
### `GET /health/admission-assessments`

### Query 参数
- `page`
- `size`
- `assessmentStatus`
- `elderId`

### 列表字段
- `assessmentId`
- `elderId`
- `agreementId`
- `assessmentStatus`
- `nurseAssessmentStatus`
- `doctorAssessmentStatus`
- `disabilityLevel`
- `startedAt`
- `completedAt`

---

## 3. 提交护士评估
### `POST /health/admission-assessments/{id}/nurse-assessment`

### Request Body
```json
{
  "nurseId": 3001,
  "assessmentFormData": {
    "bloodPressure": "120/80",
    "mobility": "需要辅助",
    "diet": "半流食"
  },
  "assessmentResult": "基础生命体征稳定"
}
```

### Response
- 返回更新后的评估状态，建议变为 `PENDING_DOCTOR_ASSESSMENT`

---

## 4. 提交医生评估
### `POST /health/admission-assessments/{id}/doctor-assessment`

### Request Body
```json
{
  "doctorId": 4001,
  "assessmentFormData": {
    "chronicDisease": "高血压",
    "riskLevel": "中"
  },
  "assessmentResult": "适合入院护理"
}
```

### Response
- 返回更新后的评估状态，建议变为 `PENDING_SUMMARY`

---

## 5. 生成评估总结
### `POST /health/admission-assessments/{id}/summary`

### Request Body
```json
{
  "summary": "老人行动能力受限，需要辅助护理",
  "disabilityLevel": "SEMI_DISABLED"
}
```

### Response
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "assessmentId": 6001,
    "assessmentStatus": "COMPLETED",
    "disabilityLevel": "SEMI_DISABLED",
    "completedAt": "2026-05-03T18:00:00"
  }
}
```

---

## 6. 家属端查看评估结果
### `GET /health/admission-assessments/{elderId}/family-view`

### Response 字段
- `assessmentId`
- `elderId`
- `assessmentStatus`
- `summary`
- `disabilityLevel`
- `completedAt`

---

# 三、caredelivery 域接口

## 1. 医护端新增服务模板
### `POST /care-delivery/service-templates`

### Request Body
```json
{
  "templateName": "半失能老人基础护理模板",
  "serviceItems": [
    { "itemCode": "MEAL", "itemName": "送餐" },
    { "itemCode": "CLEAN", "itemName": "清洁护理" }
  ]
}
```

### Response
- 返回 `templateId`

---

## 2. 医护端查询服务模板列表
### `GET /care-delivery/service-templates`

### Query 参数
- `page`
- `size`
- `enabled`
- `templateName`

### 列表字段
- `templateId`
- `templateName`
- `enabled`
- `createdBy`
- `createdAt`

---

## 3. 医护端启停服务模板
### `POST /care-delivery/service-templates/{id}/toggle`

### Request Body
```json
{
  "enabled": true
}
```

---

## 4. 为老人创建服务清单
### `POST /care-delivery/service-plans`

### Request Body
```json
{
  "elderId": 10001,
  "templateId": 8001,
  "planItems": [
    { "itemCode": "MEAL", "itemName": "送餐" },
    { "itemCode": "CLEAN", "itemName": "清洁护理" },
    { "itemCode": "MEDICINE", "itemName": "服药提醒" }
  ],
  "effectiveDate": "2026-05-04",
  "expireDate": null
}
```

### Response
- 返回 `servicePlanId`

---

# 五、阶段二状态枚举建议

## 1. 合同签字状态
- `PENDING_SIGN`
- `SIGNED`
- `CANCELLED`

## 2. 健康评估状态
- `PENDING_NURSE_ASSESSMENT`
- `PENDING_DOCTOR_ASSESSMENT`
- `PENDING_SUMMARY`
- `COMPLETED`

## 3. 失能等级
- `FULLY_DISABLED`
- `SEMI_DISABLED`

## 4. 服务清单状态
- `DRAFT`
- `ACTIVE`
- `INACTIVE`

# 六、阶段二仍待确认的接口细节

1. 合同文件是否只保存 URL，还是要记录文件摘要/版本号
2. 签字图片是否足够，还是必须保留签字轨迹数据
3. 护士评估和医生评估表单是否可配置
4. 服务模板项是否需要单独子表而不是 JSON
5. 家属端是否能查看完整评估表单，还是只看总结结果
