# 医养结合智慧养护平台 - 阶段一前端实施清单

> 范围：阶段一前端目标是把“医护端 / 家属端 / 护理员端”最小页面骨架和两个业务闭环先跑通。

---

# 一、前端实施目标

1. 医护端保留 admin 应用并补新菜单/页面
2. service-portal 支持家属与护理员两类角色视图
3. 家属预约参观闭环可跑通
4. 护理员资质申请/状态查询闭环可跑通
5. 医护端预约审核、资质审核闭环可跑通

---

# 三、smart-nursing-home-service-portal（家属端 / 护理员端）

## 1. 角色分流
### 任务
- 基于当前用户角色决定首页
- 基于角色控制菜单可见性
- 家属与护理员共用同一个应用，不共用同一套菜单

### 建议首页
- 家属：`/journey/overview`
- 护理员：`/journey/overview`

验收标准：
- 不同角色登录后自动进入对应首页

---

## 2. 家属端页面
### 页面
- `JourneyOverviewPage`（复用现有首页入口）
- `JourneyStartPage`（复用现有服务入口页）
- `FamilyVisitReservationCreatePage.tsx`
- `MyFamilyVisitReservationsPage.tsx`

### 任务 2.1：机构介绍页
- 静态/配置化展示机构介绍
- 展示宣传视频、服务介绍、机构资质

### 任务 2.2：预约参观页
- 日期选择
- 时段加载
- 表单校验
- 提交预约

### 任务 2.3：我的预约记录页
- 列表展示
- 状态标签
- 查看审核备注

验收标准：
- 家属能发起预约
- 家属能看到审核结果

---

## 3. 护理员端页面
### 页面
- `JourneyOverviewPage`（复用现有首页入口）
- `CaregiverQualificationSubmitPage.tsx`

### 任务 3.1：资质提交页

### 任务 3.2：资质状态页

- 审核备注展示

验收标准：

- 护理员能看到审核状态变化

---

## 4. 路由与菜单

## 5. API 文件
### 任务
新增：
- `src/api/admissionApi.ts`
- `src/api/qualityApi.ts`

### admissionApi
- 查询可预约时段
- 提交预约

验收标准：

# 五、前端建议开发顺序

## 医护端
1. 路由常量
2. 菜单
3. 预约审核列表/详情
4. 护理员资质审核列表/详情
5. API 接入

## service-portal
1. 角色分流
2. 家属预约页
3. 家属预约记录页
4. 护理员资质提交页
5. 护理员资质状态页
6. API 接入

# 六、前端自测清单

## 医护端
- 能看到新增菜单
- 能进入预约审核页
- 能进入资质审核页
- 能执行审核通过/拒绝

## 家属端
- 能进入预约页
- 能提交预约
- 能查看我的预约状态

## 护理员端
- 能进入资质提交页
- 能上传资料并提交
- 能查看审核状态

---

# 七、阶段一前端完成标准

满足以下条件即可视为阶段一前端完成：

1. 医护端新增审核页面可用
2. 家属预约闭环可用
3. 护理员资质申请闭环可用
4. 角色分流与权限控制正常
5. 前后端可完成基础联调
